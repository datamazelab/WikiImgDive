About

What:

WikiImgDive (short for Wikipedia Image Diversity) is a platform that supports qualitative analysis and exploration of image usage diversity on Wikipedia across 25 languages. This system provides multiple perspectives on diversity for concepts across languages. It is also a supplement of the publication: The_Tower_of_Babel.jpg: Diversity of Visual Encyclopedic Knowledge Across Wikipedia Language Editions[1] by Shiqing He, Allen Yilun Lin, Eytan Adar, Brent Hecht.

How:

Using this tool, you can browse images that are used to describe a particular concept in 25 languages. For example, if you search for the concept Chocolate (in English for example), ImgDive will display images that are used in articles that describe this concept (in different languages), such as (Chocolate, English),(Chocolat, French), and (巧克力，Chinese). We used this tool to analyze concepts to determine which images are used uniquely, or by many editions.

We display image search results in two major ways：

  1. Images are displayed in a sortable grid, which we also refer to as the image explorer.
  2. We also present a chord diagram that provides a high-level view of image use overlap for a particular concept.

Currently, our tool supports query for 25 language editions, including: English, Ukrainian, Russian, Polish, Norwegian, Dutch, Japanese, Italian, German, Finnish, Swedish, Spanish, Chinese, Catalan, French, Portuguese, Czech, Danish, Hebrew, Hungarian, Indonesian, Korean, Romanian, Slovak, and Turkish.[2] The query is space and case sensitive, i.e., article titles need to match exactly to how they appear on the Wikipedia.

The tool is open-source and currently debugged with Chrome only. If you run into browser supporting issues or need additional help, you can file a request in our github page.

Comparison:

There are implementation differences between the large-scale data processing procedure described in our paper and this online tool. We list these differences below:

Difference Paper Version Online Version
Data Source Wikipedia Dump MediaWiki Query API
Data Format and Processing Language SQL,XML: Processed with Python Json:Processed with Javascript
Data Date 2017-06-01 Latest date available on MediaWiki Query API
Type of Files Processed: 'xcf', 'djvu', 'djv', 'jpg', 'jpeg', 'jpe', 'svg', 'pdf', 'gif', 'png', 'apng', 'bmp', 'tif', 'tiff' 'jpg', 'jpeg', 'jpe', 'gif', 'png', 'apng', 'bmp'
Template Image Filtering Procedure: We used a two-stage filter that:
  1. filter out any image used in Wikipedia templates pages(templates that directly inserts images in wikitext, such as [[infobox]], are not impacted),
  2. filter out images that have template-like usage frequency.
Our paper [1] describes the procedure and accuracy of this filter in detail. 		We used a heuristic filter that eliminates images that either :
  1. contain template-like words including: [wikipedia,commons]
  2. are smaller than 200 px in width or height.
We are working on displaying these filtered images if users want to varify the accuracy of this filter.
